## What is Vyla API?

Vyla API is a **Node.js** streaming API hosted on Hugging Face Spaces. It fans out to multiple streaming providers simultaneously, verifies every stream live, and returns only the URLs that actually work — proxied, CORS-safe, and ready to drop into any HLS player.

Pass a TMDB ID. Get back working streams, multi-language subtitles, and TMDB metadata — in a single request.

<CardGroup cols={2}>
  <Card title="Multi-Provider Fanout" icon="bolt">
    All configured providers are queried in parallel with per-source jitter, retries, and timeouts. Dead sources are silently dropped from the response.
  </Card>
  <Card title="Live Stream Verification" icon="circle-check">
    Every candidate URL is fetched and checked for a valid `#EXTM3U` header before it appears in the response. No dead links.
  </Card>
  <Card title="Built-in HLS Proxy" icon="shield">
    Stream URLs are wrapped in a server-hosted proxy that rewrites M3U8 segment and key paths. CORS issues in the browser are gone by design.
  </Card>
  <Card title="Subtitles Included" icon="subtitles">
    VTT subtitle tracks in multiple languages are automatically bundled into every movie and TV response, or available standalone.
  </Card>
  <Card title="Download Links" icon="download">
    Direct download links for movies and TV episodes with quality labels and file sizes.
  </Card>
  <Card title="Health Monitoring" icon="heart-pulse">
    A dedicated health endpoint probes every provider in real time and reports per-source latency and status.
  </Card>
</CardGroup>

## How It Works

<Steps>
  <Step title="Request arrives">
    Your app sends a request with a TMDB ID (and optionally season + episode for TV).
  </Step>
  <Step title="Parallel provider fanout">
    All active providers are queried simultaneously. Each source applies random jitter to stagger upstream requests, then retries on failure up to its configured limit.
  </Step>
  <Step title="Live verification">
    Every URL returned by a provider is fetched and checked for a valid HLS playlist. Anything that doesn't respond with `#EXTM3U` is dropped.
  </Step>
  <Step title="Proxy wrapping">
    Surviving URLs are wrapped in the server proxy path. M3U8 segment and encryption key URIs are rewritten so all HLS traffic flows through the same origin.
  </Step>
  <Step title="Response returned">
    Working sources, subtitle tracks, and TMDB metadata are returned as a single JSON response — ready to feed straight into HLS.js or any HLS player.
  </Step>
</Steps>

## Endpoint Overview

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/movie` | `GET` | All working sources + subtitles + metadata for a movie |
| `/api/tv` | `GET` | All working sources + subtitles + metadata for a TV episode |
| `/api/subtitles/movie/<id>` | `GET` | Subtitle tracks for a movie |
| `/api/subtitles/tv/<id>/<s>/<e>` | `GET` | Subtitle tracks for a TV episode |
| `/api/downloads/movie/<id>` | `GET` | Download links for a movie |
| `/api/downloads/tv/<id>/<s>/<e>` | `GET` | Download links for a TV episode |
| `/api/health` | `GET` | Per-provider health check with latency |
| `/api/test/<id>` | `GET` | Debug a single provider in isolation |

<Tip>
  Check which providers are currently live: [`/api/health`](https://missourimonster-vyla.hf.space/api/health)
</Tip>

## Data Models

### Source Object

Returned inside `sources[]` on movie and TV responses.

```json
{
  "source": "provider-key",
  "label": "Provider Name",
  "url": "/api?url=https%3A%2F%2F...&pp=1"
}
```

<ResponseField name="source" type="string">
  Internal provider key. Matches the keys shown in `/api/health` and `/api/test`.
</ResponseField>

<ResponseField name="label" type="string">
  Human-readable provider name displayed in your UI.
</ResponseField>

<ResponseField name="url" type="string">
  Proxied, CORS-safe HLS stream URL. Pass directly to `hls.loadSource()`. All M3U8 segment and key URIs are pre-rewritten to flow through the proxy.

  **Important:** This is a relative path. Prepend `https://missourimonster-vyla.hf.space` to get an absolute URL.
</ResponseField>

---

### Subtitle Object

Returned inside `subtitles[]` on movie and TV responses, and as the top-level array on subtitle endpoints.

```json
{
  "label": "English",
  "file": "https://sub.vdrk.site/v1/movie/550/English.vtt",
  "type": "vtt",
  "source": "v1"
}
```

<ResponseField name="label" type="string">
  Language name of the subtitle track, e.g. `English`, `Spanish`, `Arabic`.
</ResponseField>

<ResponseField name="file" type="string">
  Direct URL to a WebVTT (`.vtt`) or SRT subtitle file. Use as a `<track>` element `src`, or pass to any player's subtitle API.
</ResponseField>

<ResponseField name="type" type="string">
  Format of the subtitle file — `vtt` or `srt`.
</ResponseField>

<ResponseField name="source" type="string">
  Internal subtitle source identifier. Informational only.
</ResponseField>

---

### Download Object

Returned inside `downloads[]` on download endpoints.

```json
{
  "url": "https://...",
  "quality": "1080p",
  "size": "2.14 GB",
  "format": "MP4"
}
```

<ResponseField name="url" type="string">
  Direct download URL.
</ResponseField>

<ResponseField name="quality" type="string">
  Quality label, e.g. `1080p`, `720p`, `480p`. `"Unknown"` if the provider doesn't supply one.
</ResponseField>

<ResponseField name="size" type="string | null">
  Human-readable file size, e.g. `2.14 GB`. `null` if unavailable.
</ResponseField>

<ResponseField name="format" type="string">
  Container format, e.g. `MP4`, `MKV`. Uppercase.
</ResponseField>

## Caching

Stream results are cached in-memory with a **5-minute TTL**, keyed by `source + tmdb_id + season + episode`. Cache hits skip the provider fetch entirely and return near-instantly. The cache resets on server restart.

---

## CORS & Headers

Every endpoint returns:

```http
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS, PATCH, HEAD
Content-Type: application/json
```

Preflight `OPTIONS` requests return `204 No Content` immediately.
